Ricky Figures It Out: Simple Authorization with React-Native StackNavigator using React Navigation.

Ok so you’ve stuck around this far… now for the fun part of React-Native! 
We get to put all three of the Navigation techniques together and make a sign-in/sign-up // authorization page!

If you’re just tuning in, this is a last of three tutorials on React Navigation.
You can find parts one and two here:
Part One: React-Native TabNavigator
&
Part Two: React-Native DrawerNavigator

Ok let’s start where we left off with the DrawerNavigator:
I renamed some of the views // styled the Sidebar a bit and created a header with Native-Base so our sidebar is now included easily in each view.

Here’s the repo for that: https://github.com/cozpon/DrawerNavigator

I recommend cloning and working from it so all the filenames match up with what I will be talking about below.

Ready?
Let’s begin

If we’ve cloned everything correctly, and have xcode installed and ran npm install and react-native run-ios and kept our finger’s crossed so everything worked properly, you should see our brand new Header on the Home Screen!

Spend a few seconds poking around and playing with the Header component we are using from Native-Base. It allows you to have 3 sections (Left, Body, Right) that can be customized to hold whatever you desire. It’s very cool!

Ok, enough playing around, let’s implement our Stack Navigator.
Let’s go into our app/config/navigation.js file and import StackNavigator alongside our other navigation tools:

import { StackNavigator, TabNavigator, DrawerNavigator } from “react-navigation”;

A good practice I have found for using Stack Navigator as an authentication wall is to implement a “root navigator”, to show different root-views based on whether a user is signed in or not:

The ‘initialRouteName’ at the end is a simple ternary if the user is “SignedIn”, then our app won’t display the “sign in” Stack Navigator screen, but rather unlock the app views.
Think of it as a wall that signing in will remove. It will make more sense with further implementation.

We need to also adjust our TabNavigator to match this new SignedIn/SignedOut logic addition.
Simply renaming the export from Tab to SignedIn will allow the TabNavigator to be displayed when a user signs in.

The Drawer Navigator/Sidebar can stay named “Drawer” since we are only calling it in our TabNavigator which is only displayed when a user is signed in! 
Of course we need to adjust our main main entry point into the application to render our “SignedIn” navigator since we renamed it.
Let’s go to our app/index.js and change it to look something like this:

import React, { Component } from 'react';
import { RootNavigator } from "./config/navigation";
import { isSignedIn } from "./auth";
export default class App extends Component {
  constructor(props) {
    super(props);
    this.state = {
      signedIn: false,
      checkedSignIn: false
    };
  }
componentWillMount() {
    isSignedIn()
      .then(response => this.setState({ signedIn: response, checkedSignIn: true }))
      .catch(error => alert("Oops! Something broked"));
  }
render() {
    const { checkedSignIn, signedIn } = this.state;
    const Layout = RootNavigator(signedIn);
    if(checkedSignIn){
      return(
        <Layout />
      );
    }else{
      return null;
    }
  }
}

We are creating a “check” to see if a user is signed in, which will control what Stack Navigation view is shown. Further on we will be creating an Authentication file called auth.js that will control our signing in/singing out, as well as the logic for checking whether a user is signed in.

Ok, making sense so far? Good.
Time to create some views for our Sign In / Logout so we can show how this Stack Navigator really works.

An easy Sign-In form can be made using the package “tcomb-form-native”.
npm install --save tcomb-form-native

They have great documentation on how to use the package and customize it to your needs. For now, let’s make a simple 2 part form, Username and Password.

Using their basic setup, I’m importing t from the package and renaming their form to something simpler. Next I’m creating a User variable to call later in our render, to allow us to have 2 input forms.

I want to talk a little about what is going on in the code above.
In the Render, we are calling on our t.form <Form /> as the type {User} which we defined in the code above. You can also have the value of the form be set to the State, if you want to implement a more advanced / secure authorization process. You would then have to set the state onPress in your _handleSubmit function…
But we will keep things simple for this article and just show how to use the StackNavigator.

Take a look at the _handleSubmit function and notice that we are calling “setStorage(true)”. This will relate to our Authorization page that I will discuss in a little bit below. It is basically using React-Native’s “Async Storage” to verify a user is logged in.

The full SignIn.js code is here:

import React, {Component} from "react";
import { View, ScrollView, StyleSheet, KeyboardAvoidingView } from "react-native";
import { Button } from "react-native-elements";
import { onSignIn, setStorage } from "../auth";
import t from 'tcomb-form-native';
const Form = t.form.Form;
const User = t.struct({
  username: t.String,
  password: t.String
});
class Login extends Component {
  constructor(props){
    super(props);
    this.state = {
      value: {
        username: '',
        password: ''
      }
    }
  }
_handleSubmit = () => {
    const navigation = this.props.navigation;
    setStorage(true)
    onSignIn()
      .then(() => navigation.navigate("SignedIn"));
  }
render() {
    return(
    <ScrollView>
      <View style={styles.container}>
        <KeyboardAvoidingView behavior="padding">
          <Form
            ref='form'
            type={User}
            value={this.state.value}
          />
          <Button
            large
            backgroundColor="#e58f19"
            textStyle={{ color: "#666" }}
            fontWeight="bold"
            raised={true}
            title="SIGN IN"
            onPress={this._handleSubmit}
          />
        </KeyboardAvoidingView>
      </View>
      </ScrollView>
    );
  }
}
const styles = StyleSheet.create({
  container: {
    justifyContent: 'center',
    marginTop: 50,
    padding: 20,
  }
});
export default Login;

Now that we have that, let’s go back to our app/config/navigation.js and import it and display it as a StackNavigator:

As you can see, if “SignedOut” is set for the initialRouteName in our RootNavigator, it will display our Sign In view (which I placed above the RootNavigator code) to allow the user to sign in.

Time to create a file to store out Authorization logic.. let’s call it auth.js and make it in line with our index.js in the app folder

Our auth.js will hold all of the logic involved with Authorization and will be the basis of our checking whether a user is logged in or not.

Easy Copy/Paste of the completeauth.js file below:

import { AsyncStorage } from "react-native";
// react-native's version of local storage
export const KEY = "rickyfiguresitout";
export const onSignIn = () => AsyncStorage.setItem(KEY, "true");
// set storage to hold key as TRUE
export const setStorage = (data) => AsyncStorage.setItem('data', JSON.stringify(data));
// set storage to hold user data
export const onSignOut = () => AsyncStorage.removeItem(KEY);
//if user signs out, remove TRUE key
export const isSignedIn = () => {
  return new Promise((resolve, reject) => {
    AsyncStorage.getItem(KEY)
      .then(res => {
        if (res !== null) {
          resolve(true);
        } else {
          resolve(false);
        }
      })
      .catch(err => reject(err));
  });
};

As you can see, we are using the AsyncStorage to hold the user’s “data”.

Note: you can store user’s information in here but it’s not best practice as it is similar storing sensitive information in a web browser’s local storage.

The logic for our Authorization file is simply when a user signs in, onSignIn is called which sets our KEY in storage, as well as gives it the value “true”. onSignOut simply removes the KEY from storage.

isSignedIn function checks whether the user’s storage has our KEY.

Simple!

Now that we can log in, let’s make sure we can logout!
Since our Sidebar/Drawer Navigation we so tediously built last tutorial should render on each view now, I have replaced our “Last” button/view with a Logout button:

On click, it will take us to a very basic “logout” page in which we can SIGN OUT or re-open our sidebar to get to different views.

The full logout.js file is here:

import React, { Component } from 'react';
import { ScrollView, Button } from 'react-native';
import { onSignOut } from '../auth';
class Logout extends Component {
_handleSubmit = () => {
    let navigation = this.props.navigation;
      onSignOut().then(() => navigation.navigate("SignedOut"));
  }
render() {
    let navigation = this.props.navigation;
    return(
      <ScrollView style={{marginTop: 200}}>
    <Button
        onPress={() => navigation.navigate('DrawerOpen')}
        title="Open Sidebar"
      />
     <Button
        backgroundColor="#03A9F4"
        title="SIGN OUT"
        onPress={this._handleSubmit}
      />
    </ScrollView>
);
  }
}
export default Logout;

Also don’t forget to change your DrawerNavigator in the app/config/navigation.js file to reflect your updates!

And if you’re like me, you forgot to update your Sidebar.js file to make the Buttons direct you to the new screen / change the title on the button.

As promised, here is our app/config/navigation.js file with the updated SignedOut StackNavigator, RootNavigator and Logout screen included:

import React from "react";
import { StackNavigator, TabNavigator, DrawerNavigator } from "react-navigation";
import DrawerContent from "../views/Sidebar"
import Profile from '../views/Profile';
import Extra from '../views/Extra';
import SignIn from '../views/SignIn';
import Home from '../views/Home';
import Info from '../views/Info';
import Logout from '../views/Logout';
export const Drawer = DrawerNavigator({
  Home: {
    screen: Home,
  },
  Info: {
    screen: Info,
  },
  Logout: {
    screen: Logout,
  }
}, {
  contentComponent: DrawerContent,
  drawerWidth: 250,
  drawerPosition: 'left',
  drawerOpenRoute: 'DrawerOpen',
  drawerCloseRoute: 'DrawerClose',
  drawerToggleRoute: 'DrawerToggle',
});
export const SignedIn = TabNavigator({
  Home: {
    screen: Drawer,
  },
  Profile: {
    screen: Profile,
  },
  Extra: {
    screen: Extra,
  }
}, {
  tabBarPosition: 'bottom',
  swipeEnabled: true,
  tabBarOptions: {
    activeTintColor: '#e58f19',
    activeBackgroundColor: "#a9c3d2",
    inactiveTintColor: '#666',
    labelStyle: {
      fontSize: 22,
      padding: 12
    }
  }
});
export const SignedOut = StackNavigator({
  SignIn: {
    screen: SignIn,
    navigationOptions: {
      title: "Sign Up"
    }
  },
});
export const RootNavigator = (signedIn = false) => {
  return StackNavigator(
    {
      SignedIn: {
        screen: SignedIn,
        navigationOptions: {
          gesturesEnabled: false
        }
      },
      SignedOut: {
        screen: SignedOut,
        navigationOptions: {
          gesturesEnabled: false
        }
      }
    }, {
      headerMode: "none",
      mode: "modal",
      initialRouteName: signedIn ? "SignedIn" : "SignedOut"
    }
  );
};

So now we can Sign In, save a KEY into our AsyncStorage, check whether that key exists, and if so let the user into the app!
We can also Sign Out and remove the KEY, which will pop up our Sign In Stack Navigator form!!!
